Sample Output

When a user clicks an element, they get detailed information like this:

Window: MCP Studio Page: Main Application Element: button#add-server-btn.sidebar-btn ID: add-server-btn Classes: sidebar-btn Selector: #add-server-btn Text: Add Server Parent: li

This gives developers everything they need:

  • Window context - Which application window
  • Unique identifier - Exact CSS selector
  • Element details - ID, classes, text content
  • Hierarchy info - Parent element context

Implementation

The complete implementation is surprisingly lightweight - just 172 lines of JavaScript:

// element-debugger.js class ElementDebugger { constructor() { this.isActive = false; this.init(); } init() { // Listen for F12 to toggle debug mode document.addEventListener('keydown', (e) => { if (e.key === 'F12') { e.preventDefault(); this.toggle(); } }); // Handle element clicks in debug mode document.addEventListener('click', (e) => { if (this.isActive) { e.preventDefault(); e.stopPropagation(); this.selectElement(e.target); } }, true); } selectElement(element) { const info = this.getElementInfo(element); this.copyToClipboard(info); this.flashElement(element); this.toggle(); // Turn off debug mode } getElementInfo(element) { return `Window: ${document.title} Page: Main Application Element: ${element.tagName.toLowerCase()}${element.id ? '#' + element.id : ''} ID: ${element.id || 'none'} Classes: ${element.className || 'none'} Selector: ${element.id ? '#' + element.id : element.tagName.toLowerCase()} Text: ${element.textContent?.trim().substring(0, 50) || 'none'} Parent: ${element.parentElement?.tagName.toLowerCase() || 'none'}`; } } // Initialize the debugger new ElementDebugger();
Debug mode activated! Click any element to copy its info.
