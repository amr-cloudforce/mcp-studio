Test Flag-Style Arguments in MCP Studio

Instructions

This page contains test configurations for verifying that MCP Studio correctly handles flag-style arguments (like --flag value) in NPX commands. Follow these steps:

  1. Copy one of the JSON configurations below
  2. Open MCP Studio
  3. Click "Add Server" and select "Import" â†’ "Paste JSON"
  4. Paste the configuration and click "Load"
  5. Verify that the flag-style arguments are correctly displayed in the form

Example 1: Supabase MCP Server

{
  "mcpServers": {
    "supabase": {
      "command": "npx",
      "args": [
        "-y",
        "@supabase/mcp-server-supabase@latest",
        "--access-token",
        "your-personal-access-token"
      ]
    }
  }
}

Expected Result

After importing, the form should show:

  • Server Name: "supabase"
  • Server Type: "NPX"
  • Repository: "@supabase/mcp-server-supabase@latest"
  • Extra Arguments: "--access-token your-personal-access-token"

Example 2: Generic MCP Server with Multiple Flags

{
  "mcpServers": {
    "generic-server": {
      "command": "npx",
      "args": [
        "-y",
        "some-package@latest",
        "--api-key",
        "your-api-key",
        "--config-path",
        "/path/to/config.json",
        "--debug"
      ]
    }
  }
}

Expected Result

After importing, the form should show:

  • Server Name: "generic-server"
  • Server Type: "NPX"
  • Repository: "some-package@latest"
  • Extra Arguments:
    • "--api-key your-api-key"
    • "--config-path /path/to/config.json"
    • "--debug"

Example 3: MCP Server with Environment Variables

{
  "mcpServers": {
    "env-server": {
      "command": "npx",
      "args": [
        "-y",
        "env-package",
        "--output-format",
        "json"
      ],
      "env": {
        "API_KEY": "your-api-key",
        "DEBUG": "true"
      }
    }
  }
}

Expected Result

After importing, the form should show:

  • Server Name: "env-server"
  • Server Type: "NPX"
  • Repository: "env-package"
  • Extra Arguments: "--output-format json"
  • Environment Variables:
    • API_KEY: "your-api-key"
    • DEBUG: "true"
